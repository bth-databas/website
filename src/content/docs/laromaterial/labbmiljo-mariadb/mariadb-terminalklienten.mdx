Nu när du har installerat databasservern MariaDB så kan vi använda terminalklienten `mariadb` för att koppla upp oss mot databasservern och utföra SQL-kommandon.



## Förutsättning

Du har skapat användaren `dbadm` och kan använda den för att logga in till databasservern.



## Använd terminalklienten mariadb

Terminalprogrammet för MariaDB hamnar automatiskt i din sökväg, din PATH, så du kan starta det direkt i terminalen.

Om du vill veta var det ligger rent fysiskt på din disk så kan du skriva så här.

```bash
which mariadb
```

Starta nu terminalklienten genom att skriva in följande kommando i terminalen.

```bash
mariadb -udbadm -pP@ssw0rd
```

Din terminalklient startar och du får en prompt där du kan skriva in kommandon. Prompten ser ut så här.

```text
MariaDB [(none)]>
```



## Vilken användare är du inloggad som?

Du kan nu utföra kommandot för att visa information om din egen användare.

```bash
SELECT USER();
```

Det som visas är detaljer om den användare som du använde för at tlogga in och från vilken plats du loggade in.

När du loggar in så kommer MariaDB att verifiera att användaren finns och att lösenordet är korrekt och sedan skapa en session för dig i databasservern med de rättigheter som användaren har.

Du kan se vilken användare som MariaDB anser att du är inloggad som genom att skriva in följande kommando.

```bash
SELECT CURRENT_USER();
``` 

Det kan se ut så här.

```text
MariaDB [(none)]> SELECT CURRENT_USER();
+----------------+
| CURRENT_USER() |
+----------------+
| dbadm@%        |
+----------------+
```



## Navigera i terminalklienten

Om du skriver in ett felaktigt kommando så kan du lägga till ett `;` så avslutas inmatningen av kommandot.

Du kan använda piltangenter tillbaka/fram för att redigera det du skriver in och du kan nå föregående kommando med pil upp/ned.

Du rensar skärmen med `ctrl-l`.

Du kan avsluta genom att skriva `exit` eller genom att trycka `ctrl-d`.



## Kommandon för databasservern

Låt oss prova att skriva in ett par kommandon för att bekanta oss med databasservern.

Du kan se vilken version av databasservern som är installerad genom att skriva in ett SQL kommando.

```sql
SELECT VERSION();
```

Det kan se ut så här men eventuellt skiljer versionssnumret hos dig.

```text
MariaDB [(none)]> SELECT VERSION();
+-----------------------------------+
| VERSION()                         |
+-----------------------------------+
| 10.11.13-MariaDB-0ubuntu0.24.04.1 |
+-----------------------------------+```

Du kan nu prova att skriva en SQL-sats för att se hur det fungerar.

```sql
SELECT NOW() AS 'The local time is';
```

Det kan se ut så här.

```text
MariaDB [(none)]> SELECT NOW() AS 'The local time is';
+---------------------+
| The local time is   |
+---------------------+
| 2026-01-09 09:21:10 |
+---------------------+
```

Du kan se vilka databaser som finns i databasservern.

```sql
SHOW DATABASES;
```

Det kan se ut så här men det kan också skilja i din installation.

```text
MariaDB [(none)]> SHOW DATABASES;
+--------------------+
| Database           |
+--------------------+
| information_schema |
| mysql              |
| performance_schema |
+--------------------+
```

Då vet vi att terminalklienten, och databasservern, fungerar som de ska.



## Avslutningsvis

Nu har du bekantat dig med terminalklienten mariadb och kan koppla upp dig mot databasservern.
